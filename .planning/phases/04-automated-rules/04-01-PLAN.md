<task type="auto">
  <name>Task 2: Create rules.ts Apps Script generator module</name>
  <files>src/modules/rules.ts</files>
  <action>
  Create new module at src/modules/rules.ts with:
  
  1. `generateAppsScript(tabs: Tab[], rules: Rule[], sheetUrl?: string): string`
     - Takes user tabs, enabled rules, and optional Sheet URL
     - Produces copy-pasteable Google Apps Script as a string
     - Structure of generated script:
       a. Header comment with extension name and generation date
       b. CONFIG object with rules array (each: { label, daysOld, action, targetLabel? })
       c. Optional SHEET_URL constant (if provided)
       d. `autoCleanup()` main function that:
          - Iterates rules array
          - For each rule: builds Gmail search query `label:{name} older_than:{days}d`
          - Calls GmailApp.search() to find matching threads
          - Maps action to GmailApp method:
            * trash → thread.moveToTrash()
            * archive → GmailApp.moveThreadsToArchive(threads)
            * markRead → GmailApp.markThreadsRead(threads)
            * moveToLabel → thread.addLabel(target), thread.removeLabel(source)
          - Wraps each rule in try/catch (one failure doesn't stop others)
          - After all rules: calls logToSheet() if SHEET_URL is set
       e. `logToSheet(results)` function that appends summary rows to Google Sheet
       f. A comment at the end with trigger setup instructions
  
  2. Only include enabled rules in the generated script (filter disabled ones)
  3. Escape special characters in label names for the script string
  4. The output should be a valid, parseable JavaScript string
  </action>
  <verify>Create test file and run: npm test -- --testPathPattern=rules</verify>
  <done>generateAppsScript produces valid Apps Script string for all 4 action types</done>
</task>
